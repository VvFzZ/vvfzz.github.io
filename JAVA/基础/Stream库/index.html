不适合使用Optional值的方式

  1. optionalVal.get() 在值不存在时抛出异常
  2. ``optionalVal.get().isPresent().get().someMethod(); 没有以下方式容易处理if(value !=null ) value.someMethod()`

并行流正常工作的条件：

  • 数据都在内存中。等待数据加载是低效的
  • 流应该被高效分成若干个字部分。数组或平衡二叉树支撑的流可以很好得工作，但Stream.Iterate返回的结果不行
  • 流操作的工作量应有较大规模。工作负载不大，并行流无异议。
  • 流操作不应阻塞
