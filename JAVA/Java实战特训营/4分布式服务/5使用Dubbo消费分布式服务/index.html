    服务引用流程

    本地接口 -> 远程调用

    1. 导入服务提供者接口API和服务信息
    2. 根据API生成远程服务的本地动态代理对象
    3. 本地API调用转换成远程服务调用，返回调用结果

    为什么使用Dubbo接口进行远程访问就像在使用本地接口一样？
    通过代理机制，创建服务API代理对象，代理对象通过远程调用返回结果。
